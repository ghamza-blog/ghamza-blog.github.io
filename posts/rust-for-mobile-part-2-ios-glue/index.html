Most of our work is going to be on glue/ios

Library Target

Inside glue/ios/Cargo.toml add the library target and make the crate type a static library

1[package]
2name = "ios"
3version = "0.1.0"
4edition = "2021"
5
6[lib]
7name = "exa"
8crate-type = ["staticlib"]

Dependencies

  • exa_core; our core library
  • libc; provides all of the definitions necessary to easily interoperate with C code (or “C-like” code) on each of the platforms that Rust supports
  • cbindgen as build dependency; to generate our library header file.

We want to export our library as C interfaces so that we can use C types and C functions directly from Swift. C interoperability with Swift.

Run

1# From the project root directory
2cargo add -p ios --path ./exa_core/
3cargo add -p ios libc
4cargo add -p ios --build cbindgen

iOS Glue Setup

Build File

Create glue/ios/build.rs and add to our bindings setup

 1extern crate cbindgen;
 2use std::env;
 3use cbindgen::Language::C;
 4
 5fn main() {
 6    setup_cbindgen();
 7}
 8
 9fn setup_cbindgen() {
10    let crate_dir = env::var("CARGO_MANIFEST_DIR");
11    match crate_dir {
12        Ok(val) => {
13            cbindgen::Builder::new()
14                .with_crate(val)
15                .with_language(C)
16                .generate()
17                .expect("Unable to generate bindings")
18                .write_to_file("include/exa_native.h");
19        }
20        Err(err) => {
21            println!("Error: {}", err);
22        }
23    }
24}

Whenever we have a non-mangeled (has #[no_mangle] attribute), public, exported C function in our lib.rs or declared as a module there, the fuction signature is going to be added to glue/ios/include/exa_native.h on save.

Create glue/ios/include/module.modulemap and add the proper info to export the headers

1module Exa {
2    header "exa_native.h"
3    export *
4}

Errors

Create glue/ios/src/error.rs that will contain our errors enum to be handled later on.

1#[derive(Debug)]
2pub enum Error {
3    InvalidUtf8,
4    InvalidUint,
5}

Prelude

Prelude is going to be imported in most of the files, so we want to make it as minimal as possibile.

Inside of it we’re going to make a shortcut for the Result type.

1pub use crate::error::Error;
2
3pub type Result<T> = core::result::Result<T, Error>;

Now we can add both of them to glue/ios/src/lib.rs

1mod error;
2mod prelude;

This pattern is considered an idiomatic way in rust for handling errors.

FFI Helpers

FFI stands for foriegn function interface, we will add helpers for converting to and from C types

Create glue/ios/src/ffi_helpers.rs

 1use crate::prelude::*;
 2use libc::c_char;
 3use std::ffi::{CStr, CString};
 4
 5pub unsafe fn to_rust_str<'a>(raw_ptr: *const c_char) -> Result<&'a str> {
 6    match CStr::from_ptr(raw_ptr).to_str() {
 7        Ok(res) => Ok(res),
 8        Err(_) => Err(Error::InvalidUtf8),
 9    }
10}
11
12pub fn to_c_str(s: &str) -> Result<*mut i8> {
13    match CString::new(s) {
14        Ok(string) => Ok(string.into_raw()),
15        Err(_) => Err(Error::InvalidUint),
16    }
17}

And finally import ffi_helpers.rs inside lib.rs

1mod error;
2mod prelude;
3mod ffi_helpers;

Glue

For the gluing part, we want to import libc and exa_core and with the help of ffi_helpers we can convert back and forth between C and Rust types.

 1mod error;
 2mod ffi_helpers;
 3mod prelude;
 4
 5use exa_core::greet;
 6use libc::c_char;
 7
 8#[no_mangle]
 9pub unsafe extern "C" fn greet_person(name: *const c_char) -> *mut c_char {
10    let Ok(name) = ffi_helpers::to_rust_str(name) else {
11        panic!("Failed to convert to rust string slice");
12    };
13    let name = greet(name);
14    let Ok(name) = ffi_helpers::to_c_str(&name) else {
15        panic!("Failed to convert to char array");
16    };
17
18    name
19}

The #[no_mangle] is an important part, so on build the function name is still the same.

Now if you save, you should be able to see the exa_native.h has added a function.

1#include <stdarg.h>
2#include <stdbool.h>
3#include <stdint.h>
4#include <stdlib.h>
5
6char *greet_person(const char *name);

Building the Project

Add these commands to the Makefile

 1ios_targets = aarch64-apple-ios x86_64-apple-ios aarch64-apple-ios-sim
 2android_targets = armv7-linux-androideabi i686-linux-android aarch64-linux-android x86_64-linux-android
 3lib_name = exa
 4framework_name = Exa
 5
 6android-setup:
 7	@for target in ${android_targets} ; do \
 8		rustup target add $$target ; \
 9	done
10
11ios-setup:
12	cargo install cargo-lipo
13
14	@for target in ${ios_targets} ; do \
15		rustup target add $$target ; \
16	done
17
18ios: ios-clean ios-build ios-framework
19
20ios-build:
21	@echo "Building for iOS..."
22
23	@for target in ${ios_targets} ; do \
24		cargo build -p ios --release --target $$target ; \
25	done
26
27	@lipo -create \
28		target/x86_64-apple-ios/release/lib${lib_name}.a \
29		target/aarch64-apple-ios-sim/release/lib${lib_name}.a \
30		-output target/lib${lib_name}_sim.a
31
32ios-framework:
33	@xcodebuild -create-xcframework \
34		-library target/lib${lib_name}_sim.a \
35		-headers glue/ios/include/ \
36		-library target/aarch64-apple-ios/release/lib${lib_name}.a \
37		-headers glue/ios/include/ \
38		-output target/${framework_name}.xcframework
39
40ios-clean:
41	@cd target && rm -rf ${framework_name}.xcframework

Note: Please make sure to use tabs rather than spaces, if we use spaces Makefile will throw this error: *** missing separator. Stop.

And now run make ios

ios-build is going to loop over the ios targets that we specified, and build the project targeting them. Then using lipo we’re going to create .a files

And finally we want to bundle the .a files inside an xcframework so we can use it in our ios app.

Using the Library

Adding XCFramework

Navigate to Frameworks, Libraries, and Embedded Content

Choose Add Other... > Add Files...

Navigate to Exa.xcframework (under target/) and Click open

Using XCFramework

We’re going to import Exa and use greet_person, simple enough.

 1import SwiftUI
 2import Exa
 3
 4struct ContentView: View {
 5    var body: some View {
 6        VStack {
 7            Button(action: {
 8                Exa.greet_person("Ghamza").map {
 9                    print(String(cString: $0))
10                }
11            }) {
12                Text("Greet")
13            }
14        }
15        .padding()
16    }
17}
18
19struct ContentView_Previews: PreviewProvider {
20    static var previews: some View {
21        ContentView()
22    }
23}

After using Exa.greet_person we’re going to receive a C char pointer, so instead of unsafly unwrapping we use .map, if we have a valid result than we can unwrap it and print it on the console.

Finally, after pressing Greet you should be able to see the log message in xcode’s output console.
